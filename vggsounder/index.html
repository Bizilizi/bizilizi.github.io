The problem

Audio-visual foundation models are supposed to understand both what they hear and what they see. But with the standard VGGSound [1] benchmark, two basic questions are surprisingly hard to answer.

1

Does the model use audio and video in a balanced way?

Joint audio-visual scores hide whether a model actually listens or just looks. Without per-modality ground truth, a vision-only shortcut looks identical to true multi-modal understanding.

2

A clip has many labels — but in which modality does each one exist?

Most videos contain several classes, and many are only audible or only visible. VGGSound [1]'s single label per clip can't say where a class appears, or even that the others are there.

VGGSounder re-annotates the VGGSound [1] test set as multi-label, with explicit per-modality and meta-label annotations, so both questions become measurable.

VGGSounder vs VGGSound

Same videos, richer ground truth. VGGSounder re-annotates the VGGSound [1] test set so you can see not just what is in a clip, but where — in audio, vision, or both.

VGGSound
  • ✗ One single label per clip
  • ✗ No modality information
  • ✗ ~48% of samples are modality-misaligned
  • ✗ No meta-labels for confounders
  • ✗ Co-occurring classes counted as errors
  • ✓ Has both train and test splits
VGGSounder
  • ✓ Multi-label: every co-occurring class
  • ✓ Per-label audible / visible / both
  • ✓ Modality-aligned subsets you can filter to
  • ✓ Meta-labels: background music, voice-over, static image
  • ✓ Synonym & superclass merging
  • ✗ Test split only (re-annotates VGGSound [1]'s test set)

This unlocks: modality profiling modality confusion controlled, confounder-free evaluation

What is modality confusion?

From audio only, the model predicts “male speech, man speaking” — correct — and never mentions a horse. But in the multi-modal setting it stops predicting “male speech, man speaking” and starts predicting “horse clip-clop” instead — the wrong label. Adding a modality overwrote a correct answer rather than improving it. Toggle the input below to see the swap.

Abstract

The VGGSound [1] dataset is commonly used to benchmark audio-visual classification, but it suffers from incomplete labelling, partially overlapping classes, and misaligned modalities, which distort evaluations of auditory and visual capabilities. We introduce VGGSounder, a comprehensively re-annotated, multi-label test set that extends VGGSound [1] and is designed to evaluate audio-visual foundation models. VGGSounder adds detailed per-label modality annotations, enabling precise modality-specific analyses, and we reveal model limitations by measuring the performance degradation that occurs when a second input modality is added — our new modality confusion metric.
